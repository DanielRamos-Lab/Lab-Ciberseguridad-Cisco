💀 2.2 Ataque #1: Backdoor Puerto 1524

Backdoor Ingreslock Trivial
Objetivo172.16.10.50 (Metasploitable)
Puerto1524/tcp
HerramientaNetcat (nc)
Acceso obtenidoroot

Ejecución

nc 172.16.10.50 1524

root@metasploitable:/# whoami
root
root@metasploitable:/# id
uid=0(root) gid=0(root) groups=0(root)
⚠️
Análisis

Este es un backdoor intencional dejado en Metasploitable. En un servidor real, representa una falla crítica de seguridad. Un atacante obtiene acceso root inmediato sin necesidad de credenciales ni exploits complejos.

💀 2.5 Ataque #4: Escaneo Windows 11

Reconocimiento de Workstation Windows Fácil
Objetivo172.16.30.100 (Windows 11)
HerramientaNmap
ResultadoPuertos SMB expuestos
nmap -sV -p 135,139,445,3389,5985 172.16.30.100

PORT     STATE  SERVICE       VERSION
135/tcp  open   msrpc         Microsoft Windows RPC
139/tcp  open   netbios-ssn   Microsoft Windows netbios-ssn
445/tcp  open   microsoft-ds?
3389/tcp closed ms-wbt-server
5985/tcp closed wsman
📝
Hallazgos

Puertos SMB (135, 139, 445) abiertos cuando el Firewall está desactivado. RDP (3389) y WinRM (5985) cerrados. En un escenario real, los puertos SMB podrían usarse para enumerar shares, usuarios y potencialmente explotar vulnerabilidades como EternalBlue.

🛡️ ETAPA 3: BLUE TEAM

Detección, Prevención y Monitoreo de Amenazas

🔒 3.1 ACL - Bloqueo de Puertos Peligrosos

Access Control List en RT-EDGE ✓ Implementado

Bloqueo de puertos vulnerables identificados durante el reconocimiento:

ip access-list extended PROTECT_DMZ
 remark === PERMITIR GESTION ===
 permit tcp 172.16.40.0 0.0.0.255 any eq 22
 permit icmp any any
 remark === BLOQUEAR PUERTOS PELIGROSOS A DMZ ===
 deny tcp any host 172.16.10.50 eq 1524 log    ! Backdoor
 deny tcp any host 172.16.10.50 eq 512 log     ! rexec
 deny tcp any host 172.16.10.50 eq 513 log     ! rlogin
 deny tcp any host 172.16.10.50 eq 514 log     ! rsh
 deny tcp any host 172.16.10.50 eq 445 log     ! SMB
 deny tcp any host 172.16.10.50 eq 139 log     ! NetBIOS
 deny tcp any host 172.16.10.50 eq 6667 log    ! IRC
 deny tcp any host 172.16.10.50 eq 6697 log    ! IRC SSL
 remark === PERMITIR TODO LO DEMAS ===
 permit ip any any

interface GigabitEthernet0/1
 ip access-group PROTECT_DMZ in

Verificación

nc 172.16.10.50 1524
(UNKNOWN) [172.16.10.50] 1524 (ingreslock) : No route to host

✓ Puerto 1524 bloqueado exitosamente

📚 4.2 Lecciones Aprendidas

1️⃣
Segmentación de red es crítica

La separación de VLANs permite aplicar controles específicos por zona. El atacante en VLAN 40 tuvo que atravesar el router para llegar a la DMZ, permitiendo la aplicación de ACLs.

2️⃣
Defensa en profundidad

Ningún control es suficiente por sí solo. La combinación de ACLs + Port Security + IDS + Logging proporciona múltiples capas de protección y detección.

3️⃣
Monitoreo es esencial

Sin logging centralizado y análisis de tráfico, los ataques pasan desapercibidos. Syslog, Snort y Wireshark proporcionan visibilidad del tráfico malicioso.

4️⃣
Servicios innecesarios = superficie de ataque

Metasploitable demuestra cómo servicios obsoletos o mal configurados (backdoors, versiones vulnerables) facilitan el compromiso total del sistema.
